# PRD: Tmux AI Kanban TUI 交互界面

## 引言

基于已完成的基础CLI功能，构建一个可交互的TUI看板界面，让用户能够用键盘导航所有AI panel，实时预览内容，一键attach到目标pane。

## 目标

- 提供沉浸式键盘驱动的交互体验
- 左侧列表展示所有AI panel，右侧实时预览选中pane的内容
- 支持智能刷新（仅在内容变化时刷新）
- 数字快捷键(1-9) + 搜索过滤(/)快速定位目标
- 按Enter一键attach到对应tmux pane

## 功能需求

### FR-1: 布局规范
- FR-1.1: 屏幕分为左右两栏，左侧占40%，右侧占60%
- FR-1.2: 左侧顶部有标题栏"AI Panels"
- FR-1.3: 右侧顶部显示当前选中pane的完整路径(session:window.pane)
- FR-1.4: 底部有统一的状态栏，高度固定为1行

### FR-2: 键盘映射
| 按键 | 功能 |
|------|------|
| `↑/↓` 或 `j/k` | 上下移动选择 |
| `1-9` | 跳转到对应序号pane |
| `/` | 进入搜索模式 |
| `Esc` | 退出搜索模式/取消操作 |
| `Enter` | Attach到选中pane |
| `r` | 强制刷新 |
| `q` 或 `Ctrl+C` | 退出TUI |
| `+/-` | 翻页（pane过多时） |

### FR-3: 智能刷新策略
- FR-3.1: 检测频率：pane列表每60秒扫描一次
- FR-3.2: 内容检测：选中pane的buffer每2秒检查一次哈希
- FR-3.3: 仅哈希变化时触发重渲染
- FR-3.4: 提供`--no-refresh`参数关闭自动刷新

### FR-4: 搜索功能
- FR-4.1: 支持模糊匹配session名、window名、目录、分支名
- FR-4.2: 搜索不区分大小写
- FR-4.3: 搜索结果实时更新列表
- FR-4.4: 无匹配时显示"No panels found"提示

### FR-5: 数据展示格式
- FR-5.1: AI类型用emoji标识：🟣Claude 🔵Codex 🟢Kimi
- FR-5.2: 状态用●(活跃)/○(空闲)表示
- FR-5.3: Session/Window/Pane显示在一列，格式为`sess:win.pane`
- FR-5.4: 工作目录显示最后两级路径，如`~/proj/kanban`
- FR-5.5: Git分支显示为`main@a1b2c3(+3)`格式

## 非目标

- 不实现鼠标支持（纯键盘驱动）
- 不实现预览内容的语法高亮（保持原始tmux输出）
- 不实现pane内容的历史回溯（只显示当前buffer）
- 不实现跨session的pane移动/重组织
- 不实现AI对话的智能总结（保持原始内容）

## 技术考量

### 性能要求
- 启动时间 < 1秒
- 列表切换延迟 < 50ms
- 内存占用 < 100MB（即使50+ pane）

### 依赖项
- textual >= 0.41.0
- 复用现有的 tmux_client, detector, git_info 模块

### 文件结构
```
src/tmux_ai_kanban/
├── tui/
│   ├── __init__.py
│   ├── app.py          # Textual App主类
│   ├── widgets/
│   │   ├── __init__.py
│   │   ├── panel_list.py    # 左侧列表组件
│   │   ├── preview.py       # 右侧预览组件
│   │   ├── status_bar.py    # 底部状态栏
│   │   └── search_box.py    # 搜索输入框
│   └── styles.tcss     # Textual CSS样式
```

### 风险与缓解
| 风险 | 缓解措施 |
|------|----------|
| 大量pane时性能下降 | 实现虚拟滚动，只渲染可见区域 |
| tmux capture-pane权限问题 | 优雅降级，显示"无法捕获"提示 |
| 内容变化检测不准确 | 同时使用时间戳+内容哈希双重校验 |
| Textual版本兼容问题 | 锁定版本 >= 0.41.0 |
